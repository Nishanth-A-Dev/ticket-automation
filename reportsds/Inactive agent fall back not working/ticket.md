Hi,
Thank you for reaching out.
The Inactive Agent Fallback will only work when the assigned agent is marked as inactive. In such cases, the system automatically reassigns the conversation to another available agent.
However, if the agent is still active but does not respond, this fallback will not be triggered.
To handle this scenario, you can use the Handover Fallback rule. This allows conversations to be reassigned when a bot assigns a conversation to an agent and the agent does not respond within a defined timeframe—essentially acting as a secondary reassignment.
You can configure this by navigating to:

Settings → WhatsApp Channel → Configurations → Agent Fallback Management → Handover Fallback Rule.

As we can see the conversation has been resolved by the chatbot and the customer initiated a new conversation in this case the agent fallback won't work. Since it is considered as a new conversation. Agent 

Handover Fallback Rule:
When an existing conversation moved from Chatbot to Agent or one agent to another agent. If the hand over agent is not replied in a particular period of time. The conversation will be then assigned to a new agent. 

Inactive Agent Fallback Rule:
If an agent in gallabox became inactive. That agent's conversation will be assigned to a new agent.

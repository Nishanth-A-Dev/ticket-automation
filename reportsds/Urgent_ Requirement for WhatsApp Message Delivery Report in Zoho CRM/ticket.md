Hi Team,

At the moment, we are unable to take up this feature implementation from our end due to current priorities. However, this requirement is technically possible, and you can configure it directly from your side using the steps below.
Step 1: Enable the Webhook in GallaboxPlease navigate to:
Gallabox Settings → Webhook
Enable the event:
Message.WA.Status.Received
Once enabled, you will receive responses in the following sample format whenever a WhatsApp message status is updated:
{
  "id": "699c1ff62ff8eed316870224",
  "status": "read",
  "timestamp": "1771839478",
  "errors": [],
  "message": {
    "recipient_id": "918778030538"
  },
  "recipient_id": "918778030538",
  "localMessageId": "045ca4de-63bf-4596-a1dc-5d3517757999"
}
Step 2: Zoho ConfigurationTo ensure records remain unique in Zoho:
Use localMessageId as the unique identifier field in Zoho.

Map recipient_id to the Customer/Lead Phone field in Zoho CRM.

You can use the Zoho Upsert API to either create a new record or update an existing one:
POST
https://www.zohoapis.com/crm/v6/{module_api_name}/upsert
The Upsert API will:
Create a record if it does not exist

Update the record if it already exists (based on the unique field)

Important NoteAny API calls made to Zoho (such as the Upsert API) may incur API usage charges as per Zoho’s billing policy. We recommend reviewing your available API limits before implementation.
Since we are currently unable to take up this request from our side, we request you to configure the workflow directly within Zoho as outlined above.
Please let us know if you need clarification on the webhook payload structure.

Message.WA.Status.Received
Once enabled, you will receive responses in the following sample format whenever a WhatsApp message status is updated:
{
  "id": "699c1ff62ff8eed316870224",

  "localMessageId": "045ca4de-63bf-4596-a1dc-5d3517757999"
}
Step 2: Zoho ConfigurationTo ensure records remain unique in Zoho:
Use localMessageId as the unique identifier field in Zoho.

Map recipient_id to the Customer/Lead Phone field in Zoho CRM.

You can use the Zoho Upsert API to either create a new record or update an existing one:
POST
https://www.zohoapis.com/crm/v6/{module_api_name}/upsert
The Upsert API will:
Create a record if it does not exist

Update the record if it already exists (based on the unique field)
